# perfdog-service-demo-v2
このデモは、PerfDog を用いた性能計測の自動化を素早く立ち上げたい方向けのサンプル実装です。コメントは英語と日本語の両方を併記しており、日本のチームでも安心して読み進められるよう整備しています。

## 動作要件
1. Python 3 をインストール（[公式サイト](https://www.python.org/) から取得。推奨は Python 3.10）。
2. Python ライブラリをインストール（`pip install -r requirements.txt` 推奨）。主要依存は次のとおりです:
   - `grpcio` (1.48.2)
   - `protobuf` (4.25.1)
3. PerfDog アカウントを作成し、後述の手順でトークンを取得しておくこと。

## セットアップ手順
1. [トークンを申請する](https://perfdog.qq.com/article_detail?id=10144&issue_id=0&plat_id=2)
2. [PerfDogService をダウンロード](https://perfdog.qq.com/perfdogservice)
3. `config.py` を作成し、トークンと PerfDogService のパスを設定する
   ```bash
   cp config.example.py config.py
   ```
   ```python
   # config.py / your machine-local secrets
   SERVICE_TOKEN = '-'    # PerfDog から発行されたトークン
   SERVICE_PATH  = '-'    # PerfDogService バイナリの絶対パス
   ```
   > `config.py` は `.gitignore` 対象です。トークンや個人マシン固有のパスが
   > リポジトリへ混入しないように設計されています。
4. CLI ユーティリティで端末・アプリ情報を確認する
   ```bash
   # 端末一覧 / Device list
   python cmds.py getdevices
   # アプリ一覧 / App list
   python cmds.py getapps <device_id>
   # システムプロセス一覧 / System process list
   python cmds.py getsysprocesses <device_id>
   # 当端末がサポートする性能指標 / Supported metrics for this device
   python cmds.py gettypes <device_id>
   # ネットワークテンプレート（プリセット + 保存済み）/ Network templates (preset + saved)
   python cmds.py getpresetnetworktemplate
   # PerfDogService を停止 / Stop PerfDogService
   python cmds.py killserver
   ```
   > Windows / Xbox を計測する場合、上記コマンドは「管理者として実行」した
   > ターミナルから起動してください。

5. 計測パラメータの設定例

### モバイル端末（Android / iOS）の性能計測
```python
# 正しい端末 ID と、計測対象アプリのパッケージ名を設定する。
# 端末 ID とアプリ一覧は cmds.py の "getdevices" / "getapps" で取得可能。
# サポート指標は "gettypes" で確認できる。
# 指標リファレンス:
#   https://perfdog.qq.com/article_detail?id=10210&issue_id=0&plat_id=2
# 1 つのスクリプトプロセスから複数端末を並行計測したい場合は、
# run_test_app をマルチスレッドで複数回起動すれば良い。
device = service.get_usb_device('-')
run_test_app(
    device,
    package_name='-',
    types=[
        perfdog_pb2.FPS,
        perfdog_pb2.FRAME_TIME,
        perfdog_pb2.CPU_USAGE,
        perfdog_pb2.MEMORY,
    ],
    dynamic_types=[
        (perfdog_pb2.GPU_COUNTER, 'GPU General'),
        (perfdog_pb2.GPU_COUNTER, 'GPU Stalls'),
    ],
)
```
> 詳細は `test.py` を参照。

### PC（Windows）の性能計測
```python
# 計測対象プロセスの PID と、レンダリングに使われている DirectX バージョンを指定する。
# Windows のプロセス一覧は cmds.py で取得可能。
# types に None を渡すと、当端末で既に有効化されている指標がそのまま使われる。
# 指標リファレンス:
#   https://perfdog.qq.com/article_detail?id=10210&issue_id=0&plat_id=2
# 並行計測したい場合は run_test をマルチスレッドで起動する。
pid = 30876
dx_version = perfdog_pb2.AUTO
run_test(
    device,
    pid=pid,
    dx_version=dx_version,
    types=[
        perfdog_pb2.FPS,
        perfdog_pb2.FRAME_TIME,
        perfdog_pb2.WINDOWS_CPU,
        perfdog_pb2.WINDOWS_MEMORY,
    ],
    enable_all_types=True,
)
```
> 詳細は `test_windows.py` を参照。

### Android / iOS の弱ネットワーク計測
```python
# 正しい端末 ID と、計測対象アプリのパッケージ名を設定する。
# 端末・アプリ・テンプレート一覧は cmds.py から取得可能。
# 自作テンプレートはスクリプト内 create_customized_template() を参照。
# 指標の意味:
#   https://perfdog.qq.com/article_detail?id=10240&issue_id=0&plat_id=1
# 並行計測したい場合は run_test_app をマルチスレッドで起動する。
device = service.get_usb_device('-')
templates = create_templates()
run_test_app(device, package_name='-', templates=templates)
```
> 詳細は `test_network.py` を参照。

### Windows の弱ネットワーク計測
```python
# 計測対象プロセスの PID を指定する。
# プロセス一覧やテンプレート一覧は cmds.py から取得可能。
# 自作テンプレートはスクリプト内 create_customized_template() を参照。
# 指標の意味:
#   https://perfdog.qq.com/article_detail?id=10240&issue_id=0&plat_id=1
# 並行計測したい場合は run_test をマルチスレッドで起動する。
templates = create_templates()
run_test(device, pid, templates=templates)
```
> 詳細は `test_network_windows.py` を参照。

### PlayStation 5 / Xbox の性能計測
```python
# コンシューマ機の追加（多くの場合は初回のみ必要）。
# 正しい端末 ID と、計測対象アプリのパッケージ名を設定する。
# 端末・アプリ・指標は cmds.py で取得可能。
# types=None の場合は、当端末で既に有効化されている指標がそのまま使われる。
service.add_remote_play_station_device(ip_address="192.168.0.0")
service.add_remote_xbox_device(ip_address="192.168.0.0", password="test")
device = service.get_wifi_device('-')
run_test(
    device,
    package_name='-',
    types=[perfdog_pb2.FPS, perfdog_pb2.FRAME_TIME],
)
```
> 詳細は `test_console.py` を参照。

6. `test.py` / `test_windows.py` / `test_network.py` / `test_console.py` を編集して実行する。
+ 必要な性能指標を有効化したり、自身の自動化テストロジックをスクリプト内に組み込むことができます。

> `test_windows.py`（Windows アプリ計測）と `test_console.py`（Xbox アプリ計測）は、
> 「管理者として実行」したターミナルから起動する必要があります。

## ご利用上の注意
- 計測が不要になったら、必ずテストを停止してください。**停止しない場合、課金が継続します。**
- PerfDogService は、Python スクリプトが終了した後もバックグラウンドでデータ収集を続けることがあります。
  終了させる場合は `python cmds.py killserver` を必ず実行してください。
- `config.py` には `SERVICE_TOKEN` 等の機密情報が含まれます。
  本リポジトリでは `.gitignore` 対象としてあり、誤って公開リポジトリへ push されないようになっています。
